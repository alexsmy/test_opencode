## Что сделано

### 1. Multi-language (10 языков)
Добавлена полноценная поддержка 10 языков в `mail_weather_agent.py`:

**`_WEATHER_DESC`** — описания погоды (WMO 0-99) для:
- en, ru, zh (были) + hi, es, ar, fr, bn, pt, id (добавлены)

**`_LOCALIZED`** — строки интерфейса (~14 ключей: humidity, wind, feels_like, pressure, cloud, forecast, precipitation, gusts, error, weather_in и др.) для всех 10 языков.

**`_VALID_LANGS`** — `{"en", "zh", "hi", "es", "ar", "fr", "bn", "pt", "ru", "id"}`

### 2. Поле `"lang"` в JSON-запросе
- `parse_weather_request()` теперь парсит `"lang"` из JSON
- Если поле отсутствует → `"en"` (было `"ru"`)
- Если невалидное → `"en"` (fallback)
- `handle_weather_message()`: приоритет lang из запроса > lang из конфига > "en"

### 3. Полная локализация ответа (critical fix)
**Проблема**: шаблон `response_format` содержал жёстко зашитые подписи (`"Weather in {location} now:"`, `"💧 Humidity:"` и т.д.) + greeting/signature брались из сохранённого конфига (английские).

**Решение** (3 последовательных фикса):

1. **Плейсхолдеры-подписи**: 
   - Добавлены `{weather_in}`, `{humidity_label}`, `{wind_label}`, `{cloud_label}`, `{pressure_label}`
   - Заполняются из `_loc(key, lang)` при форматировании
   - Все LANG_PRESETS (config.py + mail_agent.html) обновлены на единый шаблон

2. **Fallback для старых шаблонов**:
   - Если в сохранённом `response_format` нет label-плейсхолдеров → `format_weather_response()`
   - `format_weather_response()` строит ответ полностью через `_loc()` — 100% локализация

3. **Greeting/Signature из LANG_PRESETS[lang]**:
   - Если `lang` из запроса != `cfg["language"]` → имя, приветствие и подпись берутся из `LANG_PRESETS[lang]` (игнорируются сохранённые английские)
   - Если `lang` == `cfg["language"]` → используются сохранённые пользователем (кастомные)

**Итог**: при `{"location": "Paris", "lang": "fr"}` ответ полностью на французском:
```
Salut! 👋 C'est Stella.
Météo à Paris maintenant:
💧 Humidité: ...
Ta Stella. 🌟
```
