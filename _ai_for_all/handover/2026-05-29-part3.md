# Handover: 29.05.2026 (часть 3 — финал) — Mail Agent v3: Погодный агент Стелла

## Время
~3 часа. Полный цикл: план → реализация → тесты → пуш на Render → отладка → фиксы → превью.

## Что сделано

### 1. Ролевая система
- В per-inbox конфиг добавлено поле `role: "auto_reply" | "weather_agent"`
- Worker (`mail_worker.py`) диспетчеризирует по роли:
  - `auto_reply` → старый flow (save, forward, reply, mark_read)
  - `weather_agent` → `handle_weather_message()`, затем forward_to_telegram, mark_read

### 2. Модуль `services/mail_agent/mail_weather_agent.py`
- **parse_weather_request(body)** — ищет JSON `{"location": "...", "mode": "..."}` в теле письма
- **get_weather_data(location, mode, lang)** — Open-Meteo Geocoding + Forecast API
- **format_weather_response(data, cfg)** — читаемый ответ (greeting, погода, подпись)
- **format_weather_json(data)** — машинный JSON для интеграций
- **get_help_message(cfg)** — если письмо не JSON, Стелла объясняет формат
- **handle_weather_message(client, msg_id, body, cfg)** — главная точка входа

### 3. Настройки Стеллы в конфиге
Секция `weather_agent` в per-inbox конфиге:

| Поле | Тип | Дефолт |
|------|-----|--------|
| `name` | string | "Стелла" |
| `greeting` | string | "Привет! 👋 Это {name}." |
| `signature` | string | "Твоя {name}. 🌟" |
| `help_text` | string | Полный help (JSON examples) |
| `include_json` | boolean | true |
| `language` | string | "ru" |

### 4. Веб-интерфейс
- **Роль**: выпадающий список (Автоответчик / Погодный агент)
- **Блок Стеллы**: показывается только при выборе weather_agent:
  - Имя агента, язык (ru/en)
  - Приветствие, подпись, help-текст (редактируемые поля)
  - Чекбокс "Включать JSON"
- **Live preview**: автоматически обновляется при изменении полей, показывает пример ответа для Лондона

### 5. Локализация (ru/en)
- `_WEATHER_DESC[lang][code]` — описания погоды на русском и английском
- `_LOCALIZED[lang][key]` — все строки формата (humidity, wind, feels_like, etc.)
- Ветер: универсальные N/NE/E/SE/S/SW/W/NW

### 6. Исправленные баги
- **Блок Стеллы не виден до сохранения** → добавлен `toggleWeatherSettings()` в `toForm()` + `fillFormForInbox()` + событие инбокса
- **Telegram не приходит от Стеллы** → добавлен `await forward_to_telegram(parsed, client)` перед return
- **Help-текст ломался** из-за `{` в JSON → заменён `.format()` на `.replace("{name}", name)`
- **Заголовок v2** → убран

### 7. Тесты
- 17 тестов weather_agent:
  - parse_weather_request: simple, within_text, forecast, all, invalid mode, no JSON, empty, no location
  - format_weather_response: current, error, forecast
  - format_weather_json: success, error
  - get_help_message: contains Стелла + location
  - handle_weather_message: success (replied), no match (help → read), API error (read)
- Всего: 48 тестов mail agent + weather agent, зелёные ✅
