## ПЛАН НА СЛЕДУЮЩУЮ СЕССИЮ: Mail Agent Web Config UI

### Задача A: API-эндпоинты (`routers/mail_agent_api.py`)

Создать новый файл `routers/mail_agent_api.py` с 4 эндпоинтами:

**A1. `GET /api/mail-agent/config`**
- Прочитать текущий активный конфиг (объединение `config/mail_agent.json` + `data/mail_agent_config.json` override)
- Вернуть JSON со всеми полями

**A2. `PUT /api/mail-agent/config`**
- Принять JSON-тело с новыми настройками
- Сохранить в `data/mail_agent_config.json`
- Вызвать `reload_config()` из `services/mail_agent/config.py`, чтобы обновить активный конфиг в памяти
- Вернуть `{"ok": true}`

**A3. `POST /api/mail-agent/config/sync-to-cloud`**
- Прочитать `data/mail_agent_config.json`
- Запушить напрямую в GitHub через существующий `_github_push_file()` в `synchronization/mail_agent/config.json`
- НЕ вызывать `signal_sync_needed()` — это ручная операция
- Вернуть `{"ok": true}`

**A4. `POST /api/mail-agent/config/restore-from-cloud`**
- Скачать `synchronization/mail_agent/config.json` из GitHub через `_github_download_text()`
- Если есть — сохранить в `data/mail_agent_config.json` и вызвать `reload_config()`
- Вернуть `{"ok": true}`

### Задача B: Веб-страница (`templates/mail_agent.html`)

Создать HTML-страницу `/mail-agent` со следующими элементами:

**Чекбоксы (вкл/выкл):**
- `auto_reply_enabled` — автоответ
- `forward_to_telegram` — пересылка в Telegram
- `auto_mark_as_read` — пометка прочитанным
- `auto_delete_after_process` — удалять после обработки
- `save_attachments_to_disk` — сохранять вложения

**Текстовые поля:**
- `inbox_email` — email ящика
- `poll_interval_seconds` — интервал опроса (сек)
- `max_body_length_telegram` — макс. длина тела в TG (символов)
- `reply_template` — шаблон автоответа (textarea, многострочный)
- `blocked_senders` — заблокированные отправители (textarea, по одному на строку)

**Кнопки:**
- **"Сохранить настройки"** → PUT `/api/mail-agent/config`
- **"Синхронизировать в облако"** → POST `/api/mail-agent/config/sync-to-cloud`
  - Отображать статус: когда последний раз синхронизировано, есть ли несохранённые изменения
- **"Восстановить из облака"** → POST `/api/mail-agent/config/restore-from-cloud`

**Визуальный стиль:**
- Единый стиль с существующими страницами (vault.html, keepalive.html)
- JS-логика: загрузка текущего конфига при открытии страницы, отправка форм без перезагрузки
