### 2. Обработка ошибок (7 типов)
- **invalid_location** — мусор в location
- **invalid_coordinates** — координаты вне -90..90 / -180..180  
- **not_found** — город не найден геокодингом
- **api_error** — ошибка HTTP/API Open-Meteo
- **no_weather_data** — нет данных для локации
- **parse_error** — JSON decode error
- **rate_limit** — превышен лимит запросов

### 3. Pre-валидация
- `_parse_coordinates()` — парсит "lat, lon", "lat lon", "lat; lon", валидирует диапазоны
- `_validate_location()` — буквы → геокодинг, символы → reject, координаты → прямой API
- Возвращает `is_coords: bool` + `lat`/`lon` — геокодинг пропускается для прямых координат

### 4. Rate Limiter
- Скользящее окно: minute/hour/day
- `update_from_cfg()` читает `rate_limits` из конфига
- Веб-UI: 3 поля (запросов/мин, /час, /день)
- Лимиты Open-Meteo free tier: 600/мин, 5000/ч, 10000/день
