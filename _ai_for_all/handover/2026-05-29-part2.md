## Что сделано

### H1 — Карточка на главной (`static/js/hub/data/projects.js`)
- Добавлен проект "Mail Agent" с emerald-темой, ссылкой на `/mail-agent`

### H2 — Per-inbox модель конфига (`services/mail_agent/config.py`)
- Новая структура: `selected_inbox + global + inboxes: {email: ...}`
- `_inbox_defaults` + `_global_defaults` — раздельные дефолты
- `load_config()` — merge base + override, авто-миграция плоского → per-inbox
- `get_selected_inbox_config()`, `get_global_config()` — доступ к текущему инбоксу
- `save_override()` — атомарное сохранение + перезагрузка
- Новые поля:
  - `incoming_parse_max_body: 500` — макс. длина тела
  - `incoming_parse_max_subject: 100` — макс. длина темы
  - `save_history_to_disk: bool` — основная опция
  - `save_incoming: bool` — подопция (только входящие)
  - `save_attachments: bool` — подопция (только вложения)
  - `address_db_enabled: bool` — вкл/выкл БД адресов
  - `reply_templates: [{name, text}]` — массив шаблонов

### H3 — API эндпоинты (`routers/mail_agent_api.py`)
- `GET /api/mail-agent/inboxes` — список инбоксов из AgentMail + их настройки
- `POST /api/mail-agent/inboxes` — создать инбокс (AgentMail + конфиг)
- `GET /api/mail-agent/config` — весь конфиг (new format)
- `PUT /api/mail-agent/config` — сохранить конфиг
- `POST /api/mail-agent/config/sync-to-cloud`
- `POST /api/mail-agent/config/restore-from-cloud`

### H4 — Веб-страница (`templates/mail_agent.html`)
Полная переработка, 6 блоков:
1. **Email ящик** — выпадающий список (из AgentMail) + кнопка "Создать ящик"
2. **Парсинг входящих** — инпуты max_body (500) и max_subject (100)
3. **Сохранять историю на диск** — чекбокс, при включении появляются save_incoming + save_attachments
4. **БД адресов** — чекбокс address_db_enabled
5. **Шаблоны автоответов** — выпадающий список + textarea + кнопки Добавить/Удалить
6. **Остальные** — автоответ, TG, пометка, удаление, интервал, макс. длина TG, блокированные

### H5 — БД адресов (`services/mail_agent/mail_storage.py`)
- `load_address_db()` → dict[email → record]
- `save_address_db(db)`
- `update_address_db(sender, success, status)` — обновляет last_incoming / last_success / last_status
- Файл: `data/mail_agent_addresses.json`

### H6 — Интеграция
- **Воркер** (`mail_worker.py`): читает per-inbox config, использует `save_history_to_disk / save_incoming / save_attachments`, вызывает `update_address_db()` со статусом `replied`
- **Респондер** (`mail_responder.py`): выбирает шаблон по `reply_template_name`, подставляет `{summary}` из `format_summary()`
- **MCP** (`mcp_server.py`): `_get_mail_client()` читает `selected_inbox`, `mail_status()` различает inbox/global
- **Sync** (`sync/collectors.py + restore.py`): `_collect_mail_config()` собирает config, `_restore_mail_config()` восстанавливает на старте
