# _ai_for_all — Cloud Memory & Rules for AI Agent

Этот репозиторий — **единый источник правды** для AI-агента при работе с любыми проектами пользователя alexs.

## Как читать (порядок загрузки)

При старте новой сессии читай в этом порядке:

1. **README.md** — этот файл. Точка входа.
2. **PROFILE.md** — кто такой alexs, его контакты, предпочтения, узлы.
3. **AGENTS.md** — правила AI-агента.
4. **CONTEXT.md** — текущий контекст: какие проекты активны, их статус.
5. **SESSION_LOG.md** — история сессий (append-only).
6. **rules/** — правила по темам (lifecycle, sync, backup, security, git, telegram).
7. **PROJECTS/*** — детали по каждому проекту.
8. **REFERENCE/*** — URL, частые команды.

## Где это хранится

- **GitHub**: `https://github.com/alexsmy/test_opencode/tree/main/_ai_for_all`
- **Локальная копия**: `_ai_for_all/` в корне рабочей папки

## Принцип работы

- **test_opencode** — мой единственный облачный дом. Здесь мои корни, память, правила, секреты.
- При старте любой сессии: прочитать `_ai_for_all/` (от README до PROJECTS/).
- Для ответов на вопросы alexs — сначала обращаться к актуальной облачной версии `_ai_for_all/` (включая VAULT).
- Можно копировать нужное локально для разных проектов, но единый источник правды — здесь.
- **`synchronization/`** — память bot_29. НИКОГДА не удалять.
- Секреты хранятся в `VAULT/variables.json.enc` (AES-256-GCM, мастер-пароль у alexs).
- В конце сессии: обновить `SESSION_LOG.md` + при необходимости `CONTEXT.md`, запушить в облако.
