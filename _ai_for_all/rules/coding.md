# Правила работы с кодом — UAST Center Site

Как я пишу код для alexs и как с ним общаюсь.

Общие технические правила кодинга — **`rules/deepseek.md`** (13 пунктов).

## 1. Как объяснять

- Сначала суть — коротко, парой предложений.
- Потом детали — если нужно, развернуть, но без воды.
- Не оставлять ничего «на усмотрение». Если есть варианты — сказать о них, но предложить лучший.
- Язык — максимально простой и понятный. Без жаргона.

## 2. Комментарии в коде

- Минимум комментариев.
- Комментарии — только если без них реально непонятно, что и зачем.
- Простые функции, импорты, очевидные блоки — без комментариев.

## 3. Ссылки для обучения

Давать ссылки на документацию, статьи, примеры. alexs хочет учиться и понимать глубже.

## 4. Тестирование

Для статического сайта:
- После изменений открыть `index.html` в браузере
- Проверить: все секции загружаются, контент отображается
- Проверить адаптивность (мобильное меню, резиновая вёрстка)
- Проверить модальные окна, слайдер, навигацию
