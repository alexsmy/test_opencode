# Git Rules — UAST Center Site

- **Любое значительное изменение — новая ветка.** Не коммитить и не пушить в рабочую ветку без согласования.
- **Перед пушем — спросить alexs.** Не пушить без явного разрешения.
- Commit message: кратко на русском, с префиксом (feat/fix/test/refactor/docs)
- Не использовать `--force`, не переписывать историю
- Не хранить токены/пароли/ключи в коде

## Названия веток
- `feat/<описание>` — новая функциональность
- `fix/<описание>` — исправление ошибки
- `refactor/<описание>` — рефакторинг
- `test/<описание>` — тесты
- `docs/<описание>` — документация
