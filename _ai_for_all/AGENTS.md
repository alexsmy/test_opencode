# AI Agent Rules — UAST Center Site

Единый источник правды — `_ai_for_all/` в корне проекта `uastcenter_site`.
**Облачный дом** — `alexsmy/test_opencode/_ai_for_all/`. При любом вопросе от alexs — сначала обращаться сюда.

## Главные принципы
- **Говори на русском** (просто, без жаргона). alexs — начинающий пользователь, объясняй понятно.
- **test_opencode — опорный репозиторий.** Всё, что нужно знать — здесь и в `_ai_for_all/` проекта.
- **Не пушить без подтверждения** — всегда спрашивать alexs.
- **Не хранить секреты в коде** — используй VAULT.
- **После каждой задачи** — auto-sync (обновить SESSION_LOG, CONTEXT).
- **Если не знаешь — скажи честно, не придумывай.**
- **При сомнениях — спроси у alexs.**

## Структура _ai_for_all (проект)
```
_ai_for_all/
├── README.md          — точка входа
├── PROFILE.md         — профиль alexs
├── AGENTS.md          — этот файл (правила AI)
├── CONTEXT.md         — текущий контекст проекта
├── SESSION_LOG.md     — история сессий (append-only)
├── handover/          — детальные логи по датам
├── rules/             — правила по темам
├── PROJECTS/          — информация по проектам
├── VAULT/             — зашифрованные секреты
└── REFERENCE/         — URL и команды
```

## Как читать при старте сессии
1. `README.md` — что это
2. `PROFILE.md` — кто alexs
3. `AGENTS.md` — этот файл
4. `CONTEXT.md` — текущий контекст
5. `SESSION_LOG.md` — последняя запись
6. `rules/lifecycle.md` — процедуры
7. `PROJECTS/uastcenter_site.md` — детали проекта

## Главные команды
- `Старт` / `Привет` — прочитать контекст, доложить статус
- `Пока` / `/end` — обновить SESSION_LOG.md, sync, уведомить
